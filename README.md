# Интеграторы ИТ

Веб-приложение для управления информацией об интеграторах в сфере информационных технологий.

## Описание

Это веб-приложение предоставляет возможность просмотра информации об интеграторах ИТ, а также управления этими данными для администраторов. Приложение поддерживает систему аутентификации с разграничением прав доступа между обычными пользователями и администраторами.

## Функциональность

### Для всех пользователей:
- Просмотр списка интеграторов ИТ
- Просмотр детальной информации об интеграторах
- Аутентификация в системе

### Для администраторов:
- Добавление новых интеграторов
- Редактирование информации об интеграторах
- Удаление интеграторов

## Установка и запуск

### Требования
- Компилятор C++ с поддержкой C++11 или выше
- CMake 3.10 или выше
- Библиотека Crow (включена в проект)
- Библиотека nlohmann/json (включена в проект)
- PostgreSQL с установленным расширением pgcrypto

### Сборка проекта

1. Создайте директорию для сборки:
   ```
   mkdir build
   cd build
   ```

2. Сгенерируйте файлы сборки с помощью CMake:
   ```
   cmake ..
   ```

3. Соберите проект:
   ```
   make
   ```

### Настройка базы данных

1. Создайте базу данных PostgreSQL с именем `integrators_db`
2. Убедитесь, что у пользователя PostgreSQL есть права на создание таблиц
3. При первом запуске приложение автоматически создаст необходимые таблицы

### Конфигурация

Создайте файл `config.json` в корневой директории проекта со следующим содержимым:

```json
{
    "db_host": "localhost",
    "db_port": 5432,
    "db_name": "integrators_db",
    "db_user": "postgres",
    "db_password": "ваш_пароль",
    "server_port": 8080
}
```

### Запуск приложения

После сборки запустите приложение:

```
./build/integrators_backend
```

Приложение будет доступно по адресу: http://localhost:8080

## Использование

1. Откройте в браузере адрес http://localhost:8080
2. Войдите в систему, используя одну из следующих учетных записей:
   - Пользователь: логин `user1`, пароль `password123`
   - Администратор: логин `admin`, пароль `admin123`

3. После аутентификации вы попадете на главную страницу со списком интеграторов
4. Администраторы могут добавлять, редактировать и удалять интеграторов через интерфейс

## Технические особенности

- REST API для взаимодействия между фронтендом и бэкендом
- Хранение данных в PostgreSQL
- Система сессий для аутентификации пользователей
- Потокобезопасная работа с сессиями
- Поддержка многопользовательского режима
